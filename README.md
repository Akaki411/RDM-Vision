# RDM-Vision

Программа распознавания **DataMatrix** кодов с видеопотока RTSP и gRPC камер.

---

## Подготовка

Для компиляции программы потребуется:
- [rustup](https://rustup.rs/)
- [Docker](https://docs.docker.com/get-docker/)

### Docker

RTSP камеры обрабатываются через Docker-контейнер `linuxserver/ffmpeg`. Для установки можно воспользоваться Docker Compose:

```sh
docker compose up
```

При первом запуске образ будет скачан автоматически.

## Настройки

Настройки сервиса хранятся в файле config.json, при первом запуске он создается со стандартными значениями. Вот назначение параметров:

``` json
{
  "cameras": [                    - массив камер
    {
      "type": string,             - протокол подключения камеры (rtsp/grpc)
      "id": string,               - уникальный ID камеры
      "url": string,              - адрес камеры (только для RTSP)
      "endpoint": string,         - адрес сервиса камеры (только для gRPC)
      "fps": float,               - ограничение кадров в секунду
      "transport": string,        - транспорт RTP-потока: tcp/udp/auto (только для RTSP)
      "reconnect_delay_ms": int,  - пауза перед повторным подключением при обрыве
      "read_timeout_ms": int,     - таймаут ожидания кадра от камеры
      "enabled": bool             - включена ли камера
    }
  ],
  "normalization":                - параметры нормализации перед обработкой
  {
    "target_size": int,           - размер стороны кадра под вход модели
    "grayscale": bool,            - переводить ли кадр в градации серого
    "contrast": float             - коэффициент усиления контраста
  },
  "detection":                    - настройки распознавания области кода
  {
    "model_path": string,         - путь к ONNX модели YOLO
    "input_size": int,            - размер входного тензора модели (квадрат)
    "confidence_threshold": float,- порог уверенности детекции
    "nms_threshold": float        - порог IoU для подавления пересекающихся детекций
  },
  "restore_service":              - настройка сервиса восстановления Irec
  {
    "endpoint": string,           - адрес gRPC сервиса восстановления кода
    "timeout_ms": int             - таймаут запроса к сервису восстановления
  },
  "api":                          - настройка взаимодкйствия с API сервером
  {
    "base_url": string,           - адрес сервера приёма кодов
    "code_endpoint": string,      - путь для отправки распознанного кода
    "repeat_time_ms": int,        - блокировка повторной отправки одного и того же кода
    "timeout_ms": int             - таймаут запроса к серверу
  },
  "pipeline":                     - настройка самого пайплайна
  {
    "channel_capacity": int       - размер очереди кадров между камерами и пайплайном
  },
  "preview": bool                 - показывать окно предпросмотра потока (для отладки)
}
```

## Сборка и запуск
Быстрый запуск:
```sh
cargo run
```
Собрать всё в бинарник:
```sh
cargo build
```

## Тестирование
Для тестирования обработки рекомендуется воспользоваться эмулятором RTSP камеры [RTSP-emu](https://github.com/Akaki411/RTSP-emu) путем загрузки в него видеофайла с DataMatrix кодами.
